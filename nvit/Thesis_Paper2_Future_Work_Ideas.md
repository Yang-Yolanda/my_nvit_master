# Paper 2 架构演进与未来展望 (Future Work)

这篇文档总结了基于“看-拼-锁” (Global Sight -> Limb Flow -> Joint Lock) 三段式物理哲学的未来模型迭代方向，非常适合作为硕士论文最后一章的核心展望内容。

## 1. 理论基础框架的再进化：“看-拼-锁”的极致延伸

目前 NViT-Hybrid (ViT+Mamba+Soft-Guided) 已经很好地解决了 Rank Collapse 的瓶颈，但从理论极限来看，依然有巨大的升维空间：

### 阶段一：Global Sight (ViT层) 的再进化 —— “只看该看的地方”
目前 L0-L7 使用的是标准 ViT，无论背景还是人体，一视同仁地进行全局 Attention。这在算力上是极其浪费的，且容易引入背景噪声。
*   **进化方案：Token Pruning (动态词元剪枝) / 稀疏感知**
    *   **设计思想**：在 L3 或 L4 加入一个轻量级的“显著性评估器”，把明显属于背景的 Patch (Token) 直接丢弃，只保留包含人体的 Token 进入后续层。
    *   **预期收益**：大幅降低计算复杂度 (FLOPs)，并且让后续的 Mamba 层能 100% 专注在人体像素上，进一步净化特征秩 (Rank)。

### 阶段二：Limb Flow (Mamba层) 的再进化 —— “双向运动学流”
目前的 Spiral Mamba 采用的是“自中心向四肢辐射 (Center-Out)”的单向螺旋。这符合大脑发出运动指令的逻辑，但是不符合“物理逆运动学 (IK)”的反作用力传递。
*   **进化方案：Bi-directional Kinematic Mamba (双向运动学 Mamba)**
    *   **设计思想**：设计双通路 Mamba。一条通路是现有的“躯干 -> 四肢” (Forward Kinematics)；另一条通路是“末端 -> 躯干” (反向传播)。在物理世界中，脚踩地或手扶墙的位置会反向强行约束膝盖和肘部的姿态。
    *   **预期收益**：解决手部/脚部被遮挡或深度模糊时的肢体僵硬问题，引入地面接触与受力反馈。

### 阶段三：Joint Lock (Guided层) 的再进化 —— “从点引导到线引导”
现在的 Soft-Guided 逻辑是用 24 个关节点 (Points) 去“采摘” 196 个 Patch 特征。虽然保留了高维信息，但它忽略了骨骼的“长度”和“方向”约束。
*   **进化方案：Bone-Vector / Part-based Guidance (骨骼向量引导)**
    *   **设计思想**：不再预测离散的点，而是预测一组“有向线段”（比如大腿骨向量、小臂向量）。用这些线段所在的区域作为 Anchor，去聚类和池化 Patch 特征。
    *   **预期收益**：彻底消除“肢体变长/变短”的回归假象 (Bone Length Violation)，极大提升需要强物理合理性的 Mesh 重建表现。

### 终极形态：时序维度的升维 (NViT-Hybrid 4D)
目前的“三段式”仍然局限在单张静态图 (Single-frame) 内，人体的运动是连续的。
*   **进化方案：Temporal Mamba 序列拼接**
    *   **设计思想**：保留帧内的 `(ViT -> Mamba -> Guided)`，但在帧与帧之间，插入一个跨帧的 Temporal Mamba。利用 Mamba 的长序列能力将过去多帧的肢体动量 (Momentum) 传递给当前帧。
    *   **预期收益**：解决单帧算法永远无法克服的“深度歧义 (Depth Ambiguity)”和“时序抖动 (Jittering)”。

## 2. 从“空间 Mamba”到真正的“骨骼运动学 Mamba” (架构顿悟)

### 当前设计的盲区 (Spatial Mamba)
现在的 Mamba 依然是 `SpatialMamba (空间 Mamba)`。它的“中心向外”螺旋，是**基于图像像素块 (Patch) 的中心**（画面二维几何中心），而不是**基于人体的生理中心（骨盆 Pelvis）**。因此，现在的 Mamba 只是在提取“人体的整体轮廓流”，还没有真正进入形变驱动的“骨骼运动流”。

### 突破性设想：Kinematic Mamba (骨骼动力学 Mamba)
既然正向运动学是“骨盆 -> 脊柱 -> 头部”以及“肩膀 -> 肘部 -> 腕部”，我们何不让 Mamba 直接沿着这 **5 条生理通路**进行序列扫描？

也就是把“三段式”的边界提前，实现从像素域到骨骼域的跨越：

*   **👁️ 阶段一：粗看全景 (ViT, Layer 0-7)**
    *   **核心动作 (Early Guide)**：在 Layer 7 结束时，立刻做一个粗热力图预测，找到 24 个关节点的大致位置，把 196 个 Patch 的特征“吸附”到这 24 个关节点上，压缩成 **24 个 Joint-Tokens**。
*   **🌊 阶段二：运动学流动 (Kinematic Mamba, Layer 8-11)**
    *   此时 Mamba 输入不再是 196 个图像块，而是 **24 个有序的关节**！
    *   **扫描路径**：按照 5 条人体通路拼接序列（如 `[骨盆, 脊柱, 脖子, 头, 左肩, 左肘, 左腕...]`）。
    *   **动态消融设计**：
        1. **正向 (Forward)**：状态从骨盆流向指尖（神经指令）。
        2. **逆向 (Inverse)**：状态从指尖流回躯干（受力反馈）。
        3. **双向 (Bi-directional)**：兼听则明。
*   **🔒 阶段三：空间死锁 (GCN, Layer 12)**
    *   Mamba 让同一条肢体连贯了，最后用 GCN 把邻近但非直连的关节锁住（防止左右手穿模），确保绝对的三维空间合理性。

### 实验规划策略建议
对于多种 Mamba 拓扑 和多种 GCN 的探索，如果全部跑完整周期（150-250 Epochs）成本过高。
*   **最佳方案（Paper 1 经验）**：采用 `ablation campaign`，每种拓扑组合先跑 **15~20 个 Epoch**。由于修改的是底层信息流拓扑，合理的架构在早期（前20个epoch）就会展现出明显更陡的 Loss 下降曲线和更快的 MPJPE 收敛。
*   提前筛出最符合运动学直觉的“最佳组合”，再倾注全部算力进行 200 Epoch 的冲刺。

这将为论文构建极其坚实的递进逻辑：从 Baseline -> Paper 2 当前版 (解决秩塌陷) -> Paper 2 理想变体 (Kinematic Mamba 从像素跨入真正的骨骼物理流形)。

## 3. GCN 与 Guided Head 的本质联系与实验验证

### Guided Head 本质上是“软引导的变种 GCN”
在您提出使用 Guided Head 之前，传统的做法（如我们的 Run 11 实验）是强行在网络末端插入一个硬性的 GCN (Graph Convolutional Network) 层。
由于 Mamba 层的输出依然是基于大范围全局感受野的 196 个 Patch Tokens，如果直接用一个全连接层或标准图卷积把它们硬缩聚成 24 个关节 Tokens，这种暴力的降维会导致严重的 **“秩塌陷 (Rank Collapse)”** 现象（特征表达能力从高维空间急剧坍缩），从而丧失了精细的空间坐标重建能力。

而我们自己设计的 **Guided Head 本质上就是一个以“物理热力图”作为注意力特征权重的变种 GCN**：
*   **不硬行降维，而是“软采摘”**：它不是无脑地全连接压缩，而是利用 24 个关节点的 2D 热力图分布，在 196 个高维空间 Patch 中进行**加权池化 (Weighted Pooling)**。
*   **底层物理意义**：这等同于在图神经网络中，直接引入了像素点到目标关节部位的“三维物理归属概率”，用这个概率矩阵来动态决定图结构的邻接关系（Adjacency Matrix），而不是让网络在黑盒里瞎猜。

### 实验论证与论文图表设计建议

为了在论文中强有力地证明我们设计的 Guided Head（变种 GCN）的优越性，我们可以直接使用之前刚跑完的真实对比实验数据：

*   **反面教材 (Hard-GCN, Run 11 实验)**：
    *   **架构**：在 Mamba 层之后硬接标准网络图卷积降秩。
    *   **现象**：利用工具诊断出特征秩发生严重塌陷（从数十降至 1.63 左右），训练过程学习遇到瓶颈。
    *   **结果**：训练到达基准 150 轮后误差停滞（MPJPE 停滞在 **170.9mm**，PA-MPJPE 停滞在 **102.76mm**）。

*   **成功进化 (Soft-Guided Head, 现运行中的 Pivot 实验)**：
    *   **架构**：用 24 个关节的热力图去引导汇聚 196 个 Patch 特征。
    *   **现象**：信息流在 196 -> 24 的转换中平滑过渡，高维表达得已保留。
    *   **结果**：仅仅续训了 25 个 Epoch（达到 Epoch 175），**MPJPE 就暴降至 154.2mm，PA-MPJPE 降至 91.37mm**。

## 4. 落地执行方案：重塑“三段式”架构消融实验 (Ablation Campaign)

您刚才的灵魂反思极其重要：“不要依赖纯理论的猜测，必须要用严谨的对比实验来证明哪种结构最好！” 

确实，**“基于像素块 (Spatial Patches)”** 和 **“基于骨骼点 (Kinematic Joints)”** 的流转方式，各自有着截然不同的物理意义和优势。为了在论文中给出一个无可辩驳的最终结论，我们设计了如下的**“快速试错与印证”**消融工作流：

### 核心论点争锋：空间连续性 vs 骨骼拓扑连续性
*   **空间派 (Spatial Mamba)**：Mamba 在 196 个图像块上游走。它可以“看到”整张图片的上下文，包括衣服的褶皱、遮挡物、甚至是背景信息。它是在做一个“全局图像感知”的解码。
*   **骨骼派 (Kinematic Mamba)**：用 Guided Head 提前把 196 个块聚合成 24 个关节，然后 Mamba 严格沿着人体的骨架相连顺序（如肩膀->手肘->手腕）流转。它的优势是获得了极其强烈的“纯粹人体物理先验”，没有任何背景噪音的干扰。

究竟是“保留更多图像细节好”，还是“提纯出纯粹骨骼拓扑好”？这就是我们这套消融实验要回答的终极问题。

### 核心对照组设计 (Topologies)
这个消融实验旨在验证这条进化的终极路径：
1. **对照组 A (基于像素的空间基线 / Current)**：
   `ViT (196块) --> Spatial Mamba (196块空间流转) --> Guided Head (提取为24点结束)` 
2. **实验组 B1 (正向运动学 Mamba + GCN 死锁)**：
   `ViT --> Guided Head (提为24点) --> Kinematic Mamba (骨盆到指尖的正向序列流转) --> Joint-GCN (利用邻接矩阵进行空间硬约束)`
3. **实验组 B2 (逆向运动学 Mamba + GCN 死锁)**：
   `ViT --> Guided Head (提为24点) --> Kinematic Mamba (指尖到骨盆的反向受力传递) --> Joint-GCN`
4. **实验组 B3 (双向运动学 Bi-Mamba + GCN 死锁)**：
   `ViT --> Guided Head (提为24点) --> Bi-Mamba (正反双向流转) --> Joint-GCN`

### 实验推进沙盘推演
为了避免盲目消耗算力（跑一个完整模型需要 2-3 天），我们采用**“快速收敛侦察法则”**：
1. **Phase 1 (Rapid Screening 快速筛选)**：编写 `run_paper2_ablation.sh`。让组合 A, B1, B2, B3 每种架构只跑 **15 个 Epoch**。由于架构涉及底层信息流，真正优越的拓扑结构（如真实的物理双向流加 GCN 防穿模约束）在极早的时期就能展现出压倒性的 MPJPE 下降坡度。
2. **Phase 2 (Full Convergence 满力冲刺)**：拿到 Phase 1 的结果，锁定基于实验数据的真正胜出者。将全部 2 张 3090 显卡倾注于这一架构，完整跑满 200 个 Epoch，拿到可以直接上顶会的 SOTA 数据。
